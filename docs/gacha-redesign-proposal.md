# Gacha 前端视觉设计评审与重设计方案

> 文档版本：v1.0
> 日期：2026-02-11
> 目标读者：前端开发者 / UI 设计师
> 范围：`frontend/pages/gachas/*`、`frontend/components/gacha/*`、`frontend/components/Gacha*.vue`、相关 CSS 与 utils

---

## 第一部分：现有设计评审

### 1.1 总体评价

当前 gacha 系统的**功能复杂度很高**（抽卡、放置挂机、期货市场、集换交易、图鉴、任务成就），但**视觉设计严重滞后于功能深度**。整套界面给人的感受更像是一个**数据管理后台**而非**游戏化交互系统**。核心问题可归纳为六大类。

---

### 1.2 问题 ①：视觉单调性（Visual Monotony）

**严重程度：★★★★★**

几乎所有容器都使用同一套样式模板：

```
rounded-3xl border border-neutral-200/80 bg-white/90 p-6 shadow-sm
dark:border-neutral-800/70 dark:bg-neutral-900/70
```

从 `GachaDrawPanel` 到 `GachaPlacementPanel` 到 `GachaTradePanel` 到 `GachaMarketPanel`，**每一个功能区域看起来几乎一样**。用户进入页面后，无法通过视觉线索快速区分"抽卡"、"市场"、"交易"这些本质不同的功能区。

**具体表现：**
- 所有 `section` 都是白色/深灰底 + 细边框 + 微阴影
- 所有 `header` 都是 `text-lg font-semibold` + 灰色副标题
- 所有内嵌面板都是 `rounded-2xl border bg-neutral-50/75 p-4`
- 所有统计瓦片都是 `rounded-lg border border-neutral-200/70 bg-neutral-50/80 px-2.5 py-2`
- 无任何功能区之间的视觉个性区分

---

### 1.3 问题 ②：信息密度过载（Information Overload）

**严重程度：★★★★★**

**index.vue 页面**将 5 个完整功能（抽卡、放置、市场、交易、历史）全部塞入一个标签页系统。每个标签页自身就是一个复杂的子应用——`GachaMarketPanel` 包含合约选择器、K线图、多空深度、开仓控制台、持仓列表、结算列表，这些内容在单个滚动面板内堆叠，没有合理的视觉呼吸空间。

**具体表现：**
- `GachaPlacementPanel`：Hero 头部 → KPI 面板（4 个指标卡）→ 组合加成 → 无色词条槽 → 槽位网格，所有内容连续堆叠
- `GachaMarketPanel`：合约选择器 → 价格面板 → 6 个统计瓦片 → K线图 → 多空深度 → 排行榜 → 开仓控制台 → 持仓列表 → 结算列表，全部在一个面板内
- `GachaTradePanel`：发布表单 → 我的挂牌 → 公共挂牌（含搜索/排序/筛选）→ 挂牌卡片网格，全部连续排列
- 每个面板内部大量使用 `text-[10px]` 和 `text-[11px]`，文字过小导致信息难以消化

---

### 1.4 问题 ③：情感设计缺失（Lack of Emotional Design）

**严重程度：★★★★☆**

Gacha 系统的核心体验是**期待感 → 开包刺激 → 收集成就感**。当前设计完全没有这种情感弧线：

- **抽卡动作**：点击"立即抽卡"按钮后，结果直接弹窗呈现，没有任何预期构建过程（如动画、音效暗示、粒子特效）
- **开包弹窗**（`GachaDrawResultModal`）：虽然有 glow 装饰，但本质是一个静态数据报表——"总张数 / 新增 / 重复"以统计面板形式呈现，缺乏戏剧性的揭示过程
- **稀有度视觉冲击**：金色卡和白色卡在卡面上几乎没有视觉差异，稀有度仅通过左上角一个小小的 Badge 文字来传达
- **收集进度**（`progress.vue`）：仅仅是一列进度条的堆叠，没有里程碑感、成就感或视觉奖励

---

### 1.5 问题 ④：卡片设计力度不足（Underwhelming Card Design）

**严重程度：★★★★☆**

`GachaCard.vue` 具备一些高级视觉效果（foil 反光、scanline 扫描线、noise 纹理），但整体**卡面设计感缺乏**：

- 卡片图片区域使用 `aspect-ratio: 3/4`、`min-height: 10.5rem`，在网格中经常被压得过小
- 稀有度的视觉差异仅体现在：左上角一个小 Badge + `affixThemeMap` 中的 CSS filter
- 金色卡没有金色边框发光；紫色卡没有紫色光晕——**稀有度不能"一眼看出来"**
- 卡片底部信息区（标题、作者、标签、持有数）挤在一起，层次不清
- 词条标签（affix chips）用小圆角 pill 呈现，放在右上角，很容易被忽略
- 缺少卡片等级/星级等游戏化视觉符号

---

### 1.6 问题 ⑤：色彩体系过于保守（Overly Conservative Color Palette）

**严重程度：★★★☆☆**

整套系统的色彩完全围绕 Tailwind 的 `neutral` 色系构建，强调色仅通过 CSS 变量 `--accent` 和 `--accent-strong` 微量注入。这对数据分析平台的主站是合适的，但对于 gacha 这种**游戏化子系统**完全不够：

- 背景永远是 `white/90` 或 `neutral-900/70`
- 所有文字都在 `neutral-500 ~ neutral-900` 之间
- 功能性颜色（emerald 表示成功、rose 表示危险）只在很小的元素上出现
- 没有任何渐变、发光、粒子等游戏化视觉元素作为页面级装饰
- 暗色模式只是简单翻转了灰度，没有利用暗色背景带来的视觉表现力

---

### 1.7 问题 ⑥：布局与响应式问题（Layout & Responsive Issues）

**严重程度：★★★☆☆**

- `GachaMarketPanel` 没有使用 grid 来分隔主内容区和侧边栏——`<aside>` 只是在 `<div class="mt-5 grid gap-4">` 内顺序排列
- `index.vue` 的工作区标签按钮使用自定义 CSS 类 `.workspace-tab-btn` 而非 UI 组件库的 `UiTabs`，导致风格不一致
- `GachaTradePanel` 的公共挂牌网格用 `max-h-[30rem] overflow-y-auto`，在大屏上浪费空间，在小屏上需要反复滚动
- 多个面板在小屏上的体验没有专门优化——复杂的表格和表单在手机上只是简单堆叠
- 对话框（Album Variant Dialog、Album Dismantle Dialog 等）都使用同样的全屏模态框模式，没有根据内容复杂度调整尺寸

---

### 1.8 附加问题

| 问题 | 说明 |
|------|------|
| CSS 硬编码值过多 | `gacha-shared.css` 中大量 `rgba(148, 163, 184, 0.24)` 等魔术数字，暗色模式通过 `html.dark` 选择器重写而非利用 CSS 变量 |
| Scoped CSS 与全局 CSS 混用 | `GachaCard.vue` 使用 `<style scoped>`，`gacha-shared.css` 是全局导入，`GachaRouteHeader.vue` 又使用 scoped——三种模式混杂 |
| 字体声明分散 | `font-family: 'Sora', ...` 在多个组件中重复声明 |
| 动画缺乏统一编排 | 各组件独立定义 transition，没有统一的 motion design tokens |
| 缺少骨架屏/加载状态 | 数据加载时只显示文字"正在加载..."，没有骨架屏或加载动画 |
| 无空状态插画 | 空列表只有灰色文字提示，缺少空状态插画或引导性 CTA |

## 第二部分：重设计方案

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| **功能分区视觉化** | 每个核心功能区（抽卡、放置、市场、交易、图鉴）拥有独立的视觉主题色和装饰语言 |
| **情感弧线设计** | 从"期待 → 揭示 → 奖励"构建抽卡情感体验；从"探索 → 收集 → 成就"构建图鉴体验 |
| **信息渐进式揭露** | 默认展示核心数据，详细信息通过展开/悬停/点击按需揭露 |
| **稀有度即身份** | 不同稀有度的卡片在所有上下文中都必须有强烈的视觉差异 |
| **暗色模式优先** | 游戏化 UI 在暗色背景下表现力更强，以暗色模式为主要设计基准 |
| **动效有意义** | 每一个动画都服务于反馈或注意力引导，不做纯装饰性动效 |

---

### 2.2 色彩体系重构

#### 2.2.1 功能区主题色

为每个功能区分配独立的主题色对，用于该区域的渐变背景、按钮强调、图标填充：

| 功能区 | 主色 | 辅色 | 用途 |
|--------|------|------|------|
| **抽卡 (Draw)** | `violet-500` | `fuchsia-400` | 卡池卡片、抽卡按钮、开包弹窗 |
| **放置 (Placement)** | `cyan-500` | `emerald-400` | 放置引擎面板、产出指标、槽位卡片 |
| **市场 (Market)** | `sky-500` | `indigo-400` | K线图、合约选择器、开仓控制台 |
| **交易 (Trade)** | `amber-500` | `orange-400` | 挂牌卡片、价格标签、交易按钮 |
| **图鉴 (Album)** | `rose-500` | `pink-400` | 图鉴卡片、变体对话框、收集进度 |
| **任务 (Missions)** | `emerald-500` | `lime-400` | 任务列表、成就面板、票券区 |

### 2.3 页面布局重构

#### 2.3.1 总体页面结构

**将 index.vue 的单页标签模式拆分为更清晰的导航结构**：

```
/gachas                → 抽卡主页（仅抽卡 + 快捷入口卡片）
/gachas/placement      → 放置控制台（独立页面）
/gachas/market         → 市场交易所（独立页面，含市场 + 交易两个 tab）
/gachas/album          → 页面图鉴（保持不变）
/gachas/missions       → 每日任务（保持不变）
/gachas/progress       → 收集进度（保持不变）
```

**理由**：抽卡、放置、市场/交易这三个功能的复杂度各自足以撑起一个独立页面。目前将它们塞入 `index.vue` 的 5 个标签页，导致每个标签页的信息密度都过高。

#### 2.3.2 每个页面的骨架布局

所有 gacha 页面共享以下骨架：

```
┌─────────────────────────────────────────────┐
│  GachaRouteHeader（重设计后）                 │
│  ┌─────────┬───────────────────────────────┐ │
│  │ 标题区   │  导航选项卡 + 快捷操作          │ │
│  └─────────┴───────────────────────────────┘ │
├─────────────────────────────────────────────┤
│  GachaStatusBar（新组件：余额 + 通知横幅）     │
├─────────────────────────────────────────────┤
│                                             │
│  页面主内容区                                 │
│  （每个页面独立设计）                          │
│                                             │
├─────────────────────────────────────────────┤
│  GachaQuickNav（新组件：底部快捷导航卡片）     │
└─────────────────────────────────────────────┘
```

#### 2.3.3 GachaRouteHeader 重设计

**当前问题**：Header 是一个巨大的玻璃态容器，信息分两栏——左侧标题，右侧 4 个 metric-panel（时区、模式、成本）。这些 metric 对用户来说不是高频信息，占据了过多空间。

**重设计方案**：

```
┌───────────────────────────────────────────────────┐
│ 〔Gacha 小 logo / 图标〕                            │
│                                                   │
│  抽卡与放置控制台                     ┌──────────┐  │
│  先抽卡，再把库存卡片...              │ 💰 12,340 │  │
│                                      │ Token    │  │
│  ┌────┐ ┌──────┐ ┌──────┐ ┌──────┐   └──────────┘  │
│  │控制台│ │任务  │ │图鉴  │ │更多 ▾│                  │
│  └────┘ └──────┘ └──────┘ └──────┘                  │
└───────────────────────────────────────────────────┘
```

关键变更：
1. 删除 4 个 metric-panel（时区/模式/成本信息移到设置或 tooltip）
2. 右侧整合 TokenBalance 的精简版（余额 + 签到按钮）
3. 导航选项卡整合到 header 底部
4. "更多"下拉包含：放置、市场、收集进度
5. Header 高度减少约 40%

### 2.4 组件重设计方案

#### 2.4.1 GachaCard 卡片重设计

这是整个系统的核心视觉元素，必须大幅改进。

**新版卡片结构**：

```
┌─────────────────────────────┐
│ ┌─────────────────────────┐ │  ← 稀有度发光边框
│ │                         │ │
│ │       卡面图片           │ │  ← aspect-ratio: 2/3，更高更沉浸
│ │                         │ │
│ │  ┌───┐           ┌───┐  │ │  ← 左上：稀有度星级图标
│ │  │ ★★│           │词条│  │ │  ← 右上：词条标签
│ │  └───┘           └───┘  │ │
│ │                         │ │
│ │  ░░░░ 底部渐变遮罩 ░░░░  │ │
│ │  卡片标题（覆盖在图片上） │ │  ← 标题放在图片内底部
│ └─────────────────────────┘ │
│                             │
│  作者 · #标签1 #标签2        │  ← 精简信息行
│                             │
│  ┌──────────────┐ ┌──────┐  │
│  │ 持有 ×3      │ │ 操作 │  │  ← 底栏
│  └──────────────┘ └──────┘  │
└─────────────────────────────┘
```

**关键改进**：

| 项目 | 当前 | 新方案 |
|------|------|--------|
| 图片比例 | `3:4`, min-h `10.5rem` | `2:3`, min-h `14rem` |
| 标题位置 | 图片下方独立区域 | 覆盖在图片底部渐变遮罩上（白色文字） |
| 稀有度指示 | 左上角小文字 Badge | 整张卡片的边框颜色 + 左上角星级图标（★~★★★★★） |
| 词条展示 | 右上角 pill 列表 | 右上角最多 2 个图标化 chip，hover 展开详情 |
| 卡片边框 | 统一 neutral | 按稀有度发光：GOLD 金色外发光，PURPLE 紫色脉冲 |
| Foil 效果 | 仅 hover 时平移 | GOLD/PURPLE 卡默认有微弱闪烁动画 |
| 底栏 | 挤在一行 | 清晰分隔：左侧信息 + 右侧操作按钮 |

**稀有度星级映射**：

```
WHITE  → ★         （1 颗灰星）
GREEN  → ★★        （2 颗绿星）
BLUE   → ★★★       （3 颗蓝星）
PURPLE → ★★★★      （4 颗紫星，微光脉冲动画）
GOLD   → ★★★★★     （5 颗金星，持续微光 + 粒子）
```

**稀有度卡片边框效果**（CSS 实现参考）：

```css

  border-color: rgba(245, 158, 11, 0.5);
  box-shadow:
    0 0 0 1px rgba(245, 158, 11, 0.3),
    0 0 20px -5px rgba(245, 158, 11, 0.4),
    0 20px 40px -20px rgba(0, 0, 0, 0.5);
}

/* PURPLE 卡脉冲动画 */
.gacha-card--PURPLE {
  border-color: rgba(168, 85, 247, 0.45);
  animation: purplePulse 3s ease-in-out infinite;
}

@keyframes purplePulse {
  0%, 100% { box-shadow: 0 0 15px -5px rgba(168,85,247,0.3); }
  50%      { box-shadow: 0 0 25px -5px rgba(168,85,247,0.5); }
}
```

#### 2.4.2 GachaDrawResultModal 开包弹窗重设计

**这是 gacha 体验的高潮时刻，必须有戏剧性。**

**新版开包流程（分阶段揭示）**：

```
阶段 1：期待构建（~1.5s）
┌──────────────────────────┐
│                          │
│    ✦ 卡包旋转动画 ✦       │
│    （带粒子特效）          │
│                          │
│    "正在开启..."          │
│                          │
└──────────────────────────┘

阶段 2：稀有度预告（~0.8s）
┌──────────────────────────┐
│                          │
│    屏幕闪光               │
│    （GOLD = 金色闪光）     │
│    （PURPLE = 紫色闪光）   │
│                          │
│    最高稀有度文字弹出      │
│    "★★★★★ 金色!"          │
│                          │
└──────────────────────────┘

阶段 3：卡片展示
┌──────────────────────────┐
│  本次获得 10 张            │
│                          │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐     │  ← 卡片逐张翻转入场
│  │  │ │  │ │  │ │  │     │     每张间隔 120ms
```

### 2.6 动效规范

#### 2.6.1 Motion Design Tokens

```css
/* 快速反馈（按钮点击、hover 状态） */
--motion-quick: 150ms cubic-bezier(0.16, 1, 0.3, 1);

/* 标准过渡（面板展开、标签页切换） */
--motion-standard: 280ms cubic-bezier(0.16, 1, 0.3, 1);

/* 强调动效（卡片入场、弹窗出现） */
--motion-emphasis: 450ms cubic-bezier(0.34, 1.56, 0.64, 1);

/* 戏剧性动效（开包揭示、稀有卡出现） */
--motion-dramatic: 800ms cubic-bezier(0.22, 1, 0.36, 1);
```

#### 2.6.2 各场景动效规范

| 场景 | 动效 | 时长 | 曲线 |
|------|------|------|------|
| 按钮 hover | `translateY(-1px)` + shadow 增强 | 150ms | ease-out |
| 卡片 hover | `translateY(-4px)` + shadow 增强 + foil 扫过 | 280ms | ease-out |
| 标签页切换 | 内容 `opacity + translateY(8px)` 入场 | 280ms | ease-out |
| 弹窗打开 | overlay `opacity` + content `scale(0.95→1) + opacity` | 350ms | spring |
| 弹窗关闭 | 反向播放 | 200ms | ease-in |
| 开包卡片翻转 | `rotateY(90→0) + scale(0.8→1)` | 450ms/张 | spring |
| 金色卡出现 | 额外暂停 300ms + `scale(1→1.15→1)` | 600ms | spring |
| 领取奖励 | 数字跳动 + 余额更新高亮 | 500ms | spring |
| 骨架屏闪烁 | `background-position` shimmer | 1.5s loop | linear |
| 槽位运转 | 微弱脉冲光环 | 3s loop | ease-in-out |

#### 2.6.3 Reduced Motion 支持

所有动效在 `prefers-reduced-motion: reduce` 下需要：
- 取消所有循环动画
- 将过渡时间缩减为 0ms 或极短
- 保留 opacity 过渡（作为视觉反馈替代）
- 开包动画直接跳到结果

### 2.8 暗色模式设计规范

**核心理念**：暗色模式不是"翻转灰度"，而是利用深色背景让色彩更突出。

#### 2.8.1 暗色模式的优势利用

- **发光效果**：稀有度卡片的 glow/shadow 在暗色背景上表现力 10x 于浅色
- **渐变背景**：功能区的主题色渐变在暗色下更醒目
- **对比度**：白色文字在深色背景上天然高对比
- **沉浸感**：暗色更适合游戏化界面的氛围

#### 2.8.2 暗色模式配色原则

| 层级 | 颜色值 | 用途 |
|------|--------|------|
| 基底 | `#030712` (gray-950) | 页面背景 |
| 表面 1 | `rgba(15,23,42,0.85)` | 主要容器 |
| 表面 2 | `rgba(30,41,59,0.65)` | 嵌套容器 |
| 表面 3 | `rgba(51,65,85,0.45)` | 内嵌面板 |
| 边框 | `rgba(100,116,139,0.25)` | 默认边框 |
| 边框强调 | 稀有度对应色 40% | 交互边框 |
| 文字主 | `#F1F5F9` (slate-100) | 标题、重要数字 |
| 文字副 | `#94A3B8` (slate-400) | 副标题、说明 |

### 2.10 技术约束与注意事项

| 约束 | 说明 |
|------|------|
| **框架** | Nuxt 4 + Vue 3 + TypeScript，不引入新框架 |
| **样式方案** | 继续使用 Tailwind CSS + 局部 CSS，不引入 CSS-in-JS |
| **UI 基础** | 继续使用 Radix Vue 无头组件 |
| **动画** | 优先使用 CSS `@keyframes` + `transition`，复杂序列可用 Vue `<Transition>` + `onEnter` 钩子 |
| **兼容性** | 支持最近 2 个版本的 Chrome/Firefox/Safari/Edge |
| **性能** | 动画使用 `transform`/`opacity` 触发 GPU 合成层；避免 `layout` 属性动画 |
| **无障碍** | 所有交互元素需要 `aria-label`；`prefers-reduced-motion` 全面支持 |
| **不改变业务逻辑** | 本次重设计仅改变视觉呈现和交互动效，不修改 composables 和 API 层 |
| **Composable 层不动** | `useGacha.ts`、`useGachaIndex.ts` 等 composable 保持不变，仅组件模板和样式变化 |
